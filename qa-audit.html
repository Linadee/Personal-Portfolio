Document 02 — Bug report
Product images are not clickable and have no hover state
SCRUM-3 · Products page · Filed April 10, 2026
Severity
High
Priority
High
Status
Open
Reporter
Lina Diaz
Environment
Chrome 124 / macOS
Component
Products page
Description
All product images on the Products page are displayed as static, non-interactive elements. Users cannot click on any product and there is no hover effect to indicate interactivity. This blocks the core product browsing flow entirely.
Steps to reproduce
  1. 1.Navigate to allspecenclosures.com
  2. 2.Click "Products" in the navigation bar
  3. 3.Hover the cursor over any product image
  4. 4.Attempt to click any product image
Expected result
Product images should display a hover effect (cursor change, highlight, or overlay) indicating interactivity. Clicking a product should navigate the user to a product detail page or initiate a quote request flow.
Actual result
No hover state is applied when cursor moves over product images. Clicking any product image produces no action — the page remains unchanged.
Impact
Users have no way to access individual product information or initiate a purchase or quote from the Products page. SCRUM-4 (product redirect test) is blocked until this defect is resolved.
